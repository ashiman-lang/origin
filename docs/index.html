How Milestones And Forecasts Work

Milestones are cumulative revenue points. M1 means revenue through cohort end + 1 month, M2 through cohort end + 2 months, M3 through cohort end + 3 months, and so on. If a milestone cutoff has already passed, the dashboard shows the actual net value. If the cutoff has not passed yet, the dashboard forecasts only the missing future tail.

What Counts As Actual

  • Closed M1/M2/M3 values use actual renewal progress per customer: M1 means first renewal, M2 means second renewal, M3 means third renewal.
  • Net means paid transaction revenue minus refunds, with dispute losses subtracted as a cohort-level proxy.
  • This avoids dropping real renewals just because Stripe retried the billing a few days later than the calendar milestone date.
  • Cohort membership and New counts still come from the customer export.

Benchmark Rates Used

  • Monthly: M1 55%, M2 27%, M3 12%, M4 10%, M5 5%, M6 3%, M7 3%.
  • Quarterly: M3 55%, M6 20%, M9 15%, M12 10%.
  • Annual: M12 40%.

How Prediction Starts

  • Each customer is assigned to a starter family from their first paid Subscription creation transaction: monthly, quarterly, or annual.
  • Guest customers (`gcus_*`) and one-time guest purchases are excluded from subscription-family prediction.
  • If the current Plan later goes blank, that customer still stays in the original starter-family denominator, so canceled starters remain included.
  • The last closed actual milestone for that family becomes the anchor retention point.

How Future Revenue Is Added

  • Later milestone retention is projected by applying the benchmark month-over-month ratio to the last actual retention (for example: if actual M1 retention = 46% and the monthly benchmark goes from M1 55% to M2 27%, then predicted M2 retention = 46% × 27% / 55% = 22.6%).
  • The projected future tail uses each priced customer’s real full renewal price, not a generic bucket price.
  • First-day upsell revenue stays inside cumulative predicted ROAS, and projected future revenue gets a 15% haircut for refunds and fees.
Worked Example: Jan 1-Jan 7

Actual closed milestones

  • Spend: €10,676. D0 actual: €4,207, M1 actual: €4,842, M2 actual: €5,353, M3 actual: €8,491.
  • Because this weekly cohort is already closed for M3, the displayed M3 is actual, not projected: €8,491 / €10,676 = 79.5%.

Starter-family denominators

  • Monthly starters: 63 with actual retentions M1 52.4%, M2 25.4%, M3 11.1%.
  • Quarterly starters: 77 with actual M3 retention 57.1%.
  • Annual starters: 9.

How M6 is projected from the last actual month

  • Monthly anchor is actual M3 retention 11.1%. Then M4 = M3 × 10%/12% = 9.3%, M5 = M4 × 5%/10% = 4.6%, M6 = M5 × 3%/5% = 2.8%.
  • Quarterly anchor is actual M3 retention 57.1%. Then M6 = M3 × 20%/55% = 20.8%.
  • The projected M6 tail uses the real renewal-price pools and then applies the 15% haircut: monthly tail €298, quarterly tail €907, annual tail €0.
  • So projected M6 = actual M3 base €8,491 + future tail €1,205 = €9,695, which is 90.8% ROAS.
